# Knight Squad Academy Jail 2
문자열 입력 시 작동하는 knight() 함수를 발견했습니다.  
문자열 길이가 30일때만 올바르게 작동합니다.  

![image6](/assets/images/KnightCTF/image6.png)  

위치가 맞으면 앞에, 틀리면 뒤 쪽 숫자에 카운트가 올라가는 시스템입니다.  

![image7](/assets/images/KnightCTF/image7.png)  


**명령어 자동화 스크립트 작성**

```python 
def check_char_positions(template, target_char):
    # 템플릿 문자열 리스트화
    template_list = list(template)
    # '1'의 인덱스들 찾기
    indices = [i for i, char in enumerate(template) if char == '1']
    
    print(f"--- '{target_char}'가 들어갈 수 있는 {len(indices)}가지 경우 ---")
    
    for idx in indices:
        temp_list = list(template) # 원본 복사
        temp_list[idx] = target_char # 해당 위치만 L로 변경
        
        # 가독성을 위해 변경된 글자만 강조하거나 표시 가능
        result_flag = "".join(temp_list)
        print(f"knight(\"KCTF{{{result_flag}}}\")")

# 현재 찾은 플래그 상황
current_progress = "1a11t1111111111011b1o111"

# 실행
check_char_positions(current_progress, 'L')
```

# Vulnerability Exploitation
HTTP 트래픽을 분석했습니다.  

```bash
tshark -r pcap2.pcapng -Y "http.request" -T fields -e ip.src | sort | uniq -c | sort -rn
```

분석 결과 의심스러운 대량 요청이 발견되었습니다.  

26172 192.168.1.104    ← 🚨 의심스러운 대량 요청!
23 192.168.1.101  
6 192.168.1.103  

192.168.1.104 활동을 추적해보았습니다.  

```bash
tshark -r pcap2.pcapng -Y "http.request and ip.src==192.168.1.104" -T fields -e http.request.uri | head -30
```

**결과** 
![image14](/assets/images/KnightCTF/image14.png) 

플러그인 스캔을 탐지해보았습니다.  
```bash
tshark -r pcap2.pcapng -Y "http.request and ip.src==192.168.1.104" -T fields -e http.request.uri | grep -i "wp-content/plugins" | head -20
```
**결과**   
`WPScan v3.8.28 (https://wpscan.com/wordpress-security-scanner)`

HTTP 객체를 추출했습니다.  

```bash
mkdir -p /tmp/http_objects
tshark -r pcap2.pcapng --export-objects http,/tmp/http_objects
find /tmp/http_objects/ -name "*readme*"
```

**결과**  
![image15](/assets/images/KnightCTF/image15.png) 

버전을 확인했습니다.  

```bash
cat /tmp/http_objects/readme.txt | grep -i "stable\|version"
```

**결과**  
![image16](/assets/images/KnightCTF/image16.png) 

이름과 버전이 플래그인 문제입니다.  

이름: Social Warfare  
버전: 3.5.2

**Flag**  
`KCTF{social_warfare_3.5.2}`

# Post-Exploitation
pcap 파일 다운로드 후 TCP 연결 통계를 확인했습니다.  

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -q -z conv,tcp 2>&1 | grep -v "^$" | head -30
```
**결과**  
포트 80이 다수 확인되었습니다.  
![image17](/assets/images/KnightCTF/image17.png) 

HTTP 요청을 분석했습니다.  

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -Y "http.request" -T fields -e tcp.dstport -e http.request.uri 2>&1 | grep -v "^Running" | head -20
```

**결과**

![image18.png](/assets/images/KnightCTF/image18.png)

192.168.1.104 에서 payload.txt 로드 시도한 흔적이 보였습니다.  
SSRF 공격으로 추측됩니다.  

다른 포트를 분석해보았습니다.  

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -q -z conv,tcp 2>&1 | grep -v "^Running" | grep -v "80 " | grep -v "Filter" | grep -v "^=" | grep -v "Frames" | grep -v "^$" | head -20
```

**결과**  
192.168.1.102 ↔ 192.168.1.104:9576 사이 300초 이상의 긴 연결이 확인되었습니다.  
![image19.png](/assets/images/KnightCTF/image19.png)

payload.txt 요청을 분석했습니다.  

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -Y "http.request.uri contains \"payload.txt\"" -T fields -e frame.number -e ip.src -e tcp.dstport 2>&1 | grep -v "^Running"
```

**결과**  
8767번 포트가 확인되었습니다.  
![image20.png](/assets/images/KnightCTF/image20.png)

8767번 포트를 탐색했습니다.  

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -Y "tcp.port == 8767" -T fields -e frame.number -e ip.src -e ip.dst -e tcp.srcport -e tcp.dstport 2>&1 | grep -v "^Running" | head -20
```

**결과**  
192.168.1.102 → 192.168.1.104:8767 http 페이로드 전달을 확인했습니다.  
![image21.png](/assets/images/KnightCTF/image21.png)

8767 http 요청을 확인했습니다.  

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -Y "tcp.port == 8767" -V 2>&1 | grep -A 5 -B 5 "payload.txt" | head -30
```

**결과**   
get 요청으로 payload.txt 다운로드하는 것을 확인했습니다. 
![image22.png](/assets/images/KnightCTF/image22.png) 
포트 8767에서 http 페이로드를 전달하고, 포트 9567에서 리버스 쉘 연결로 추정했습니다.  

두 포트 연결 통계 확인

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -q -z conv,tcp 2>&1 | grep -E "(8767|9576)" | grep -v "^Running"
```

**결과**  
연결이 확인되었습니다.  
![image23.png](/assets/images/KnightCTF/image23.png)

연결된 두 포트가 플래그인 문제입니다.  
FLAG형식: **KCTF{httpPort_revshellPort}**

**FLAG**  
`KCTF{8767_9576}`

```bash
cd /mnt/user-data/uploads && tshark -r pcap3.pcapng -q -z follow,tcp,ascii,37378 2>&1 | grep -v "^Running" > /tmp/shell_session.txt && head -200 /tmp/shell_session.txt
```

**결과**  
wp-config.php 파일을 불러오는 것을 확인했습니다.
